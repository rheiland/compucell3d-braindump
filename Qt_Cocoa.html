Trolltech blog re 4.6 on OSX 10.6

Trolltech - dev on Mac

Mantis bug report for mapping Cocoa concepts

End up trying (for Snow Leopard, OSX 10.6): 

cd (path-to)/qt-all-opensource-src-4.5.3
./configure -opensource -cocoa -framework -arch x86_64 -nomake examples -nomake demos -sdk "/Developer/SDKs/MacOSX10.6.sdk"

but during the build, I got an error related to MySQL, so I had to reinstall a 64-bit version, Mac OS X 10.5 (x86_64), from here (that's right, it's actually for 10.5, but will apparently work for 10.6, since they don't yet provide one for 10.6). Argh, I see that they now provide 5.1.40 whereas a couple days ago when I downloaded, it was 5.1.39. Once that was installed, we find: 

heiland@129-79-58-76:~/dev$ mysqld --version
mysqld  Ver 5.1.39 for apple-darwin9.5.0 on i386 (MySQL Community Server (GPL))

heiland@129-79-58-76:~/dev$ which mysqld
/usr/local/mysql-5.1.39-osx10.5-x86_64/bin/mysqld

heiland@129-79-58-76:~/dev$ file /usr/local/mysql-5.1.39-osx10.5-x86_64/bin/mysqld
/usr/local/mysql-5.1.39-osx10.5-x86_64/bin/mysqld: Mach-O 64-bit executable x86_64

and then Qt seemed to build fine - hours later :(
