OpenMP OSX

From CompuCell3D Wiki

Jump to: navigation, search

Basically, one needs to figure out how to get OpenMP working correctly on OSX. For now, we go here: http://hpc.sourceforge.net/

and download/install: 

gcc-snwleo-intel-bin.tar.gz (Snow Leopard), gfortran-snwleo-intel-bin.tar.gz (Snow Leopard gfortran only), updated April 2011 (Snow Leopard only).

You need to "sudo tar xf" each of these tarballs from "/" to have them install to the proper locations:
-->
heiland@dhcp-cs-244-118:/usr/local/bin$ ll *4.6*
-rwxr-xr-x@ 1 root  wheel  465648 Apr  3 22:33 g++-4.6*
-rwxr-xr-x@ 1 root  wheel  461472 Apr  3 22:33 gcc-4.6*
-rwxr-xr-x@ 1 root  wheel   91200 Apr  3 22:33 gcov-4.6*

heiland@dhcp-cs-244-118:/usr/local/bin$ ll gfortran*
-rwxr-xr-x@ 1 root  wheel  466512 Apr  3 21:37 gfortran*


heiland@dhcp-cs-244-118:/usr/local/include$ ll c++/4.6.0/
...

vs. the old:
$ll /usr/include/c++/
total 0
drwxr-xr-x  68 root  wheel  2312 Dec 29  2009 4.0.0/
drwxr-xr-x  68 root  wheel  2312 Dec 29  2009 4.2.1/

But beware of the frequent problem: 

g++: error: x86_64: No such file or directory
g++: error: unrecognized option ‘-arch’
g++: error: unrecognized option ‘-Xarch_x86_64’

Googling for 'unrecognized option arch osx qmake', we find:

http://stackoverflow.com/questions/6252822/how-a-recent-version-of-gcc-4-6-could-be-used-together-with-qt-under-mac-os

Had a discussion w/ Alin about him trying to use the hpc.sf.net gcc 4.6 vs. trying to build his own gcc...
